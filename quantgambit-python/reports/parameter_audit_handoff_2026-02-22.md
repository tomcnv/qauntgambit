## Core Decision Logic
- Entries are only possible after: data readiness -> global gates -> profile routing -> prediction gate -> signal generation -> confirmation/alignment -> EV/cost gates -> candidate veto -> risk -> execution.
- Exits are evaluated early in `position_evaluation`; safety exits bypass non-emergency confirmation and fee checks when risk is critical.
- Unified confirmation policy is enabled in `enforce` mode for entry and non-emergency exit (vote + confidence model with hard guards).
- Prediction path is ONNX-primary with score gate in `fallback_heuristic` mode; if ONNX is blocked/insufficient, heuristic fallback can be used depending on gate state.
- EV gate is enforced; entries must clear expected edge after conservative cost modeling and minimum net edge buffers.
